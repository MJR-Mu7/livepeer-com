# List streams

You can retrieve all `stream` objects with the same `userID`. The response will include both `stream` objects that
represent a livestream configuration (no `parentId` parameter) and stream objects that represent a single livestream session (`parentId` is the same as the `id` of the parent `stream` object). 

```bash
curl -H 'authorization: Bearer <api-key>' \
"https://livepeer.com/api/stream/"
```

### Query parameters

| JSON parameter              | Type    | Description                                                                                                                                                                                                                               |
| ----------------------------| ------- | ----------------------------------------------------------------------|
| `isActive`                  | boolean | `true` returns active `stream` objects. `false` returns idle `stream` objects. The default value for new `stream` objects is `false`. |
| `parentId`                  | number  | To return all sessions of the same `stream` object, set to the the value of the parent `stream` object` id `. To return only  "parent" `stream` objects, set the value to ???? |
| `record`                    | boolean | `true` returns `stream` objects with recording on. `false` returns `stream` objects with recording off. The default value for new `stream` objects is `false`. |
